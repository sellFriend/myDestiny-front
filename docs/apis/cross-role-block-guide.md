# 매물·주선자 상호 진입 차단 — 프론트 연동 가이드

## 개요

서비스 내 두 역할(매물·주선자)은 동시에 가질 수 없습니다. 서버에서 세 계층으로 차단합니다.

- **매물 사용자** = 어떤 주선자의 지인 목록에 등록된 사람 (`Acquaintance.friendUserId`)
- **주선자** = 자신의 지인 목록을 가진 주선자 (`Acquaintance.userId`)

---

## 차단 시점 정리

| 상황 | 엔드포인트 | 상태 코드 |
|---|---|---|
| 매물 사용자가 API 호출 | 모든 `/api/**` | `403` |
| 매물 사용자가 폼 링크 발급 시도 | `GET /api/invite` | `403` |
| 주선자가 폼 진입 시도 | `GET /form/{madamId}` | `400` |
| 이미 다른 주선자 지인인 사용자가 폼 진입 | `GET /form/{madamId}` | `400` |
| 주선자가 폼 제출 시도 | `POST /form/{madamId}` | `400` |
| 이미 다른 주선자 지인인 사용자가 폼 제출 | `POST /form/{madamId}` | `409` |

> `/form/**` 경로는 매물 사용자도 접근 가능합니다. 폼 진입·제출 로직 안에서 역할 충돌을 판단합니다.

---

## 응답 형식 주의사항 ⚠️

**403이 두 종류입니다. 필드 구조가 다릅니다.**

### 필터 단 403 (매물 사용자 → `/api/**`)

`GlobalExceptionHandler`를 거치지 않아 구조가 다릅니다.

```json
{
  "status": 403,
  "error": "Forbidden",
  "message": "매물로 등록된 사용자는 서비스를 이용할 수 없습니다."
}
```

### 서비스 단 403 / 400 / 409

```json
{
  "success": false,
  "message": "...",
  "data": null
}
```

## 케이스별 응답 및 처리

### 매물 사용자가 API 호출 (필터 단 403)

```json
{ "status": 403, "error": "Forbidden", "message": "매물로 등록된 사용자는 서비스를 이용할 수 없습니다." }
```

→ 로그아웃 처리 후 안내 페이지로 이동

---

### 매물 사용자가 폼 링크 발급 (`GET /api/invite`) → 403

```json
{ "success": false, "message": "매물로 등록된 사용자는 주선자 역할을 할 수 없습니다.", "data": null }
```

→ 폼 링크 발급 버튼 비활성화 + 토스트 메시지

---

### 폼 진입 (`GET /form/{madamId}`) 케이스별

| 상태 | `message` | 처리 |
|---|---|---|
| `200` | — | 폼 표시 |
| `400` | `본인의 폼에는 등록할 수 없습니다.` | 기존 처리 |
| `400` | `주선자는 지인으로 등록될 수 없습니다.` | message 노출 후 폼 진입 중단 |
| `400` | `이미 다른 주선자를 통해 등록되어 있습니다.` | message 노출 후 폼 진입 중단 |
| `404` | — | 기존 에러 처리 |

---

### 폼 제출 (`POST /form/{madamId}`) 추가 케이스

| 상태 | `message` | 처리 |
|---|---|---|
| `400` | `주선자는 지인으로 등록될 수 없습니다.` | message 노출 후 제출 중단 |
| `409` | `이미 다른 주선자를 통해 등록되어 있습니다.` | message 노출 후 제출 중단 |

---

## 403 구분 방법 (코드 예시)

```js
if (response.status === 403) {
  // 필터 단: success 필드 없음
  if (data.success === undefined) {
    // 매물 사용자 전체 차단
    logout(); redirect('/blocked');
  } else {
    // 서비스 단: 링크 발급 불가 등
    showToast(data.message);
  }
}
```
