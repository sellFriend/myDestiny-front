---

## 1. 작업 요약 (Summary)

- **한 줄 요약**
  - 이 작업이 무엇을 해결하거나 추가하는지 간단히 설명

---

## 2. 작업 배경 / 목적 (Background)

- 왜 이 작업이 필요했는지
- 기존 문제점 또는 요구 사항
- 기획/요청의 출처 (있다면 간단히)

---

## 3. 작업 내용 (Details)

### 3-1. 기능 개발 (Feature Development)

- **구현한 기능**
  - 어떤 기능을 추가했는지
  - 주요 동작 흐름
- **핵심 로직 / 포인트**
  - 중요한 조건, 분기, 상태 관리
  - 서버/웹 역할 분리 여부
- **영향 범위**
  - 영향 받는 페이지 / 컴포넌트 / API

---

### 3-2. 버그 개선 (Bug Fix)

- **증상 (Symptom)**
  - 사용자 또는 QA가 인지한 문제
- **원인 (Root Cause)**
  - 기술적 원인
  - 왜 발생했는지
- **해결 방법 (Solution)**
  - 어떤 방식으로 수정했는지
  - 기존 로직 대비 변경점
- **재발 방지**
  - 가드 로직, 테스트, 문서화 여부

---

### 3-3. 리팩토링 (Refactoring)

- **작업 설명**
  - 어떤 코드를 왜 정리했는지
- **변경 전 / 후**
  - 구조, 책임, 네이밍, 패턴 변화
- **개선 효과**
  - 가독성 / 유지보수성 / 성능 / 안정성
- **기능 변경 여부**
  - [ ] 있음 (사유 명시)
  - [ ] 없음 (순수 리팩토링)

---

- 변경 전 / 변경 후 비교 (있다면)
- 사용자 체감 포인트
